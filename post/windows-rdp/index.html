Windows断开远程桌面RDP后保持桌面活跃的方法

  1. 2021年10月12日
  2. 214字
  3. 1 min read

✅前言

在Windows server 2019上挂脚本时，没有外界显示器的服务器在断开远程桌面后，桌面就会自动休眠，导致脚本停止运行。

✅探索解决

解决办法无非有这几种：
1️⃣安装模拟显示器驱动，假装接入了显示器，找不到，失败！
2️⃣ 使用VNC Server或TeamViewer等其他远控方式。并不好用！

✅成功解决

如果通过%windir%\System32\tscon.exe RDP-Tcp#*NNN* /dest:console手动断开RDP的话，原桌面也不会休眠，其中NNN是当前远程桌面的session。可用下面一段脚本自动完成：

for /f "skip=1 tokens=3" %%s in ('query user %USERNAME%') do ( %windir%\System32\tscon.exe %%s /dest:console )

另存为bat文件

✅TIPS

每次断开RDP时不要关闭RDP客户端，而通过右键→以管理员身份运行来手动断开RDP。
